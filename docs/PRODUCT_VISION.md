# Station Product Vision

## Product definition

Station is a browser-first sampler groovebox for quickly transforming raw audio samples into playable pads, patterns and complete musical sketches. It combines pad performance, step sequencing and a musical Pump workflow in an interface designed like an instrument rather than a traditional desktop DAW.

## Primary promise

A user should be able to open Station, import samples, build a groove and make it move musically without understanding complex audio routing or compressor terminology.

## Core workflow

1. Start the audio engine through an intentional user gesture.
2. Import WAV samples into a 16-pad bank.
3. Play samples directly through touch, mouse or keyboard.
4. Build a 16-step Pattern Group and, when useful, create up to A–D variations.
5. Arrange short pattern clips in a practical-length Playlist, including parallel clips.
5. Select a kick source and apply Basic Pump to another track.
6. Save the project locally and reopen it later.

## Product principles

1. Station is a sampler groovebox, not a small DAW.
2. The browser is the product platform, not a temporary demo environment.
3. Fewer systems should create depth through combination, sequencing and later resampling.
4. One screen should have one primary musical purpose.
5. Important actions must remain usable on a phone or tablet.
6. Technical complexity should be hidden behind musical choices.
7. Timing stability and touch response outrank visual spectacle.
8. Features are accepted only when they shorten or deepen the path from sound to groove.
9. We do not copy the visual language or terminology of one existing device 1:1.
10. We do not design speculative architecture for a native port that may never exist.

## Primary user

The first user is a beatmaker or electronic-music creator who:

- prefers samples and rhythm over a blank multitrack timeline,
- wants fast results and a playful instrument-like workflow,
- may work on desktop, phone or tablet,
- values character and constraints,
- does not want to configure complex routing for common musical effects.

## Success criteria

The early product succeeds when:

- pad response feels immediate enough to play,
- the sequencer remains musically stable under normal UI activity,
- a new user can build a pattern without reading a manual,
- Basic Pump produces an obviously useful result with very few controls,
- the project survives closing and reopening the browser,
- the same core workflow works on desktop and real mobile browsers.

## Anti-goals

Station is not intended to provide:

- an infinite multitrack timeline,
- full MIDI composition and piano-roll editing,
- plugin hosting,
- professional vocal recording and comping,
- unlimited routing,
- dozens of conventional effects,
- video editing,
- real-time cloud collaboration,
- exact emulation of an MPC, SP-404, Digitakt or tracker.

The Playlist is a limited pattern-based arrangement system: it uses fixed one-pattern clips, not arbitrary audio clips or a full DAW timeline.
